## What works (verified by tests)

- Storage CRUD: issue → job → step rows.
- Playbook keyword + OS matching (`macos-slow` matches "my mac is slow").
- Promise-keyed dispatcher: server awaits agent result as if it were a local call.
- Async ReAct loop with **server-enforced tool whitelist** — the brain cannot escape the playbook.
- Full happy path: `POST /issues` → playbook → orchestrator → fake agent over real WebSocket → final answer in JSON.

## What does NOT work yet — code review found blockers

The keystone test uses a **fake agent** that fabricates tool results. The real agent-side tool layer was copied from RepairCraft and uses CrewAI's `@tool` decorator, which wraps functions in non-callable Tool objects. Hardware test would crash on first real dispatch.

### Blockers (must fix before Mac mini + MacBook test)

1. **Agent tools are CrewAI-wrapped.** All `agent/tools/*.py` import `from crewai.tools import tool` and decorate functions. `agent/executor/runner.py` registers them as plain callables — `await asyncio.to_thread(fn, **args)` will not invoke them correctly, and `crewai` isn't even in `requirements.txt`. Fix: strip `@tool` decorators, leave the underlying functions.
2. **Tool catalog/registry drift.** Catalog (server) advertises 6 tools; runner (agent) registers 8. L2 escalation in `playbooks/macos/slow.yaml` lists `run_safe_command`, which on the agent side is currently a broken CrewAI Tool object.

### Important (one-shot polish, recommended before hardware test)

3. **Dispatcher leak on agent disconnect.** `Dispatcher.unregister_agent` does not cancel pending futures. Mid-job disconnect hangs `/issues` for the full 60s timeout. Fix: track `job_id → machine_id`, fail those futures on disconnect.
4. **`.env.example` ships a Postgres URL** but the server defaults to SQLite. A user copying the example to `.env` will fail on first boot. Fix: comment out or default to `sqlite:///mspclaw.db`.
5. **`GeminiBrain.step` blocks the event loop.** It calls the sync `generate_content` inside `async def`. With ~8 LLM steps × multi-second calls, it serializes every in-flight `/issues`. Fix: wrap in `asyncio.to_thread`.
6. **`enduser/cli.py:25`** uses `typer.Exit("string")` — `typer.Exit` takes an int. This raises `TypeError` instead of the friendly message. Fix: `print(...)` then `raise typer.Exit(1)`.

### Out of scope for v0.1 (intentional, documented)

Postgres, multi-tenant enforcement, audit log queries, `dynamic_fix`, web portal, email/PSA intake, Windows agent, real shared-token auth.
